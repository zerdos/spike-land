<Tldr>

- The spike.land MCP worker is intentionally usable from other origins.
- Public metadata is readable without auth; authenticated runtime calls work with API keys or OAuth device-flow tokens.
- The clean default for browser integrations is `mcp.spike.land`, not the first-party proxy routes on `spike.land`.
- You can fetch tool schemas, call tools, and embed store apps into an existing product without moving your whole app onto spike.land.

</Tldr>

## The Trap Most Platforms Set

Most platforms quietly assume you will rebuild around them.

Use our UI. Use our domain. Use our SDK. Use our auth flow. Use our hosting. Before you know it, your "integration" has become a migration project — which is another way of saying it will never ship.

This is a familiar pattern in systems thinking: a reinforcing loop that benefits the platform at the expense of the user. Each dependency you add makes the next one feel inevitable, until switching costs are so high that you are locked in by gravity, not by choice.

That is not the model we want for spike.land.

The better model is simple:

**Use the runtime where it helps. Keep the rest of your stack.**

---

## What Is Actually Open

The spike.land MCP worker exposes public metadata endpoints for tools and apps.

That means you can:

- read tool definitions
- render UIs from live schemas
- inspect app metadata
- bootstrap OAuth device flow

And when you are ready to execute tools, you move to the authenticated MCP endpoint with either an API key or a device-flow token.

The important part: this works from another origin. The platform does not require you to co-host the UI on spike.land. Your app stays your app. spike.land is just the tool shelf on the wall — reach for what you need, when you need it.

---

## Why `mcp.spike.land` Matters

There are two surfaces to understand, and the distinction between them matters more than it looks:

- `spike.land` for first-party product UX
- `mcp.spike.land` for the MCP runtime itself

If you are building a browser app on another origin, call `mcp.spike.land` directly. That is the surface designed to expose wildcard CORS for MCP and public metadata.

Think of it like the difference between a restaurant's dining room and its wholesale counter. Same kitchen, different interfaces. "Same product" and "same integration surface" are not the same thing.

---

## The 5-Minute Flow

If you already have an app, the shortest path looks like this:

1. fetch `https://mcp.spike.land/tools`
2. inspect the tool you want
3. obtain an API key or OAuth token
4. send a JSON-RPC `tools/call` request to `https://mcp.spike.land/mcp`
5. render the result inside your own UI

That is it.

No iframe platform prison. No forced rewrite. No fake "integration" that is really a sales funnel dressed up as documentation.

Five steps between "I wonder if I could use this" and "it is running in my product." That is the kind of friction that respects your time.

## Why This Changes the App Store

Once the runtime is open cross-origin, the app store stops being just a store. It becomes something more interesting — a set of composable layers:

- a **discovery layer** — find what exists
- a **metadata layer** — understand what it does
- a **distribution layer** — get it to your users
- an **execution layer** — run it where you need it

An app can be listed on spike.land, embedded in another product, called by an agent, or wrapped inside a Cloudflare Worker that adds product-specific logic on top.

In systems thinking terms, the store is not a destination — it is a stock that feeds multiple flows. The same tool can flow into a dozen different products without the store needing to know about any of them. That is a much more interesting model than "here is a directory page."

## The Real Win

The real win is not CORS headers.

The real win is not having to choose between:

- using a platform
- keeping your existing product

Most platforms force that choice because lock-in is their business model. The more you depend on them, the harder it is to leave. That is a balancing loop working against you — more integration creates more dependency creates more cost to switch.

If spike.land is going to be a real MCP app store, it has to work as infrastructure, not just as a destination website. Infrastructure serves you wherever you are. Destinations make you come to them.

That is why we built the cross-origin path first.

A platform you can only use from inside its own walls is not a platform. It is a walled garden with a nicer name.
