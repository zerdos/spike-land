## Проблема, о которой никто не говорит честно

N умножить на M. Вот проблема интеграций, которая прячется у всех на виду.

N AI-клиентов. M внешних сервисов. Для каждой комбинации нужна авторизация, парсинг ответов, обработка ошибок, ограничение частоты запросов, обнаружение инструментов. Написанные с нуля, каждый раз заново.

Если у вас 5 клиентов и 10 сервисов — это 50 путей интеграции. Добавьте ещё один сервис — и вам нужно 5 новых интеграций. Поверхность растёт квадратично. Бремя поддержки следует за ней.

MCP (Model Context Protocol) сокращает проблему N-на-M до N-плюс-M. Один протокол, реализованный один раз с каждой стороны. Любой клиент общается с любым сервером. Это тот же инсайт, который сделал USB успешнее дюжины проприетарных кабелей — общий контракт побеждает кастомную проводку.

Именно на этом стандарте мы построили spike.land. Сегодня мы запускаемся.

## Что такое spike.land

spike.land — это **MCP-first AI-платформа** — реестр ИИ-инструментов, которые ваши агенты могут вызывать через единый стандартизированный протокол. Вместо того чтобы подключать отдельные API, ваш AI-клиент подключается к spike.land и мгновенно получает доступ к 80+ готовым к продакшену инструментам.

Никаких кастомных интеграций. Никакого склеивающего кода для авторизации. Никакой логики пагинации. Просто инструменты, готовые к вызову.

### 80+ инструментов, которые мы выпускаем сегодня

Наш начальный набор охватывает пять категорий:

**Chess Arena** — Полноценные соревновательные шахматы с рейтингом ELO, вызовами в реальном времени и управлением состоянием игры. Шесть инструментов. `chess_new_game`, `chess_send_challenge`, `chess_make_move`, `chess_get_board`, `chess_get_elo`, `chess_resign`.

**Image Studio** — Анализ, генерация, улучшение изображений на базе ИИ, извлечение палитр, управление альбомами и оркестрация пайплайнов. 42 инструмента, построенные на компонуемой middleware-архитектуре, описанной в [The Architecture of Scale](/blog/the-architecture-of-scale-building-tools).

**Codespace** — Выполнение кода, транспиляция и среды предпросмотра в реальном времени. Пять инструментов для запуска TypeScript и JavaScript в изолированных песочницах.

**QA Studio** — Автоматизация браузера через Playwright. Восемь инструментов для скриптового end-to-end тестирования, захвата скриншотов и записи взаимодействий.

**Utilities** — Веб-запросы, поиск, файловые операции и растущий набор инструментов общего назначения, которые компонуются между категориями.

Каждый инструмент следует одному контракту: типизированная входная схема на Zod, нарративный ответ, структурированный JSON-вывод. [Почему нарративные ответы?](/blog/mcp-explained-universal-adapter-for-ai-tools#narrative-responses-tell-stories-not-dump-data) Потому что AI-агенту нужны не просто данные — ему нужны данные, над которыми он может рассуждать в контексте.

## MCP Tool Marketplace

Мы запускаем маркетплейс spike.land одновременно с платформой.

Идея проста: если вы создали полезный MCP-инструмент, вы должны получать деньги, когда другие его используют. Мы берём на себя инфраструктуру — авторизацию, ограничение частоты, биллинг, обнаружение, соответствие протоколу. Вы занимаетесь бизнес-логикой. Вся инженерная обвязка — наша забота, не ваша.

**Разделение выручки: 70/30. Авторы инструментов получают 70%.**

Опубликованные инструменты появляются в реестре spike.land и становятся вызываемыми любым пользователем spike-cli. Биллинг основан на кредитах — пользователи платят за вызов, вы зарабатываете за каждое успешное выполнение.

Для публикации инструмента вам нужно:
1. Валидное определение MCP-инструмента (имя, описание, Zod-схема, обработчик)
2. Аккаунт разработчика на spike.land
3. Опубликованный npm-пакет или развёрнутый Cloudflare Worker

[Паттерн tool builder](/blog/the-architecture-of-scale-building-tools), который мы используем внутри — тот же, что используют авторы маркетплейса. Та же поверхность API. Тот же формат ответов. Легко для агентов обнаруживать и компоновать.

Сегодня мы открываем маркетплейс для первых партнёров. Если у вас есть инструменты для публикации, [подайте заявку на ранний доступ](https://spike.land/marketplace/apply).

## API Gateway: кредиты и BYOK

Многие из наших инструментов внутренне вызывают ИИ-модели — анализ изображений, объяснение кода, генерация контента. Мы предлагаем два способа оплаты этих вызовов моделей, потому что гибкость важна, когда вы разбираетесь, что работает.

**Bring Your Own Keys (BYOK)**

Подключите свои API-ключи OpenAI, Anthropic или Replicate в дашборде. Вызовы инструментов, которым нужна модель, маршрутизируются через ваши ключи. Ваши расходы. Ваши лимиты. Полный контроль.

**Прокси-кредиты spike.land**

Предпочитаете не управлять несколькими API-аккаунтами? Используйте прокси-кредиты spike.land. Один пул кредитов, консолидированный биллинг, без аккаунтов на каждый сервис.

- 1 кредит = 1 стандартный вызов API модели (приблизительно, зависит от модели и количества токенов)
- Кредиты переносятся ежемесячно на тарифах Pro и Business
- Покупка дополнительных кредитов по $0.10/кредит

Прокси работает как passthrough — мы не накручиваем стоимость моделей сверх конвертации кредитов. Наша маржа — на платформе и маркетплейсе, не на инференсе моделей.

## Технический стек

spike.land полностью работает на Cloudflare Workers. Никаких серверов для настройки. Никаких регионов для конфигурации. Запросы автоматически маршрутизируются к ближайшей edge-ноде. Инфраструктура уходит с дороги, чтобы инструменты могли делать свою работу.

- **Edge API**: фреймворк Hono на Cloudflare Workers
- **MCP-реестр**: Cloudflare Workers + D1 (SQLite на edge)
- **Авторизация**: Better Auth с Drizzle ORM
- **Фронтенд**: Vite + React + TanStack Router
- **Среда выполнения инструментов**: изолированные Worker-среды для каждой категории инструментов

[Полная архитектура](/docs/architecture) задокументирована. Мы строим открыто и продолжим это делать.

## Почему MCP?

Мы могли бы построить REST API. REST — привычная штука. Все знают, как сделать curl к эндпоинту.

Но REST API были спроектированы для разработчиков-людей, уставившихся в документацию. Они оптимизированы под компактный JSON, статус-коды, токены пагинации. AI-агентам нужен другой контракт — тот, что объясняет модели на естественном языке, *что* делает данная возможность, *как* её вызвать с типизированными параметрами и *что означает* ответ в контексте.

MCP — это такой контракт. Он определяет обнаружение, типизированные схемы и нарративные ответы как first-class примитивы. Когда ваш AI-агент подключается к spike.land через MCP, ему не нужны страницы документации. Он читает описания инструментов, выводит правильные вызовы и выполняет их. Протокол сам по себе является документацией.

Мы подробнее написали об этом в [MCP Explained](/blog/mcp-explained-universal-adapter-for-ai-tools). Коротко: MCP — это слой представления, а не просто транспорт. Он формирует то, как модель *думает* о ваших возможностях, а не только то, как она их вызывает.

## Что дальше

80+ инструментов — это начало, а не потолок.

**Q2 2026**: MCP Resources и Prompts. Сегодня spike.land работает только с инструментами — каждое взаимодействие — это вызов функции. Resources позволят агентам просматривать состояния игр, профили пользователей и каталоги инструментов как адресуемый контент. Prompts позволят серверам предоставлять переиспользуемые шаблоны взаимодействий.

**Q2 2026**: Публичный запуск маркетплейса. Инструменты первых партнёров начнут работу. Начнутся выплаты от разделения выручки.

**Q3 2026**: UI для композиции инструментов. Создавайте многошаговые пайплайны из существующих инструментов в визуальном редакторе. Код не нужен.

**Q3 2026**: Обмен инструментами между агентами. Публикуйте приватные инструменты внутри Business-команды, видимые только вашим агентам.

Мы создали spike.land, чтобы скучные части ИИ-интеграции исчезли. Авторизация, обнаружение, контракты инструментов, биллинг — решено. Вы пишете логику агента. Мы берём на себя остальное.
