## 没人坦诚谈论的问题

N 乘 M。这是隐藏在眼前的集成问题。

N 个 AI 客户端。M 个外部服务。每种组合都需要认证、响应解析、错误恢复、速率限制、工具发现。每次都从头写起。

如果你有 5 个客户端和 10 个服务，那就是 50 条集成路径。再加一个服务就需要 5 条新集成。面积按乘积增长，维护负担随之而来。

MCP (Model Context Protocol) 把这个 N 乘 M 的问题降到了 N 加 M。一个协议，每一端实现一次。任何客户端都能和任何服务器通信。这和 USB 胜过十几种专有线缆的道理一样——共享契约胜过定制接线。

这就是我们构建 spike.land 所基于的标准。今天，我们正式上线。

## spike.land 是什么

spike.land 是一个 **MCP 优先的 AI 平台**——一个 AI 驱动工具的注册中心，你的 agent 可以通过单一标准化协议来调用。不用逐个对接 API，你的 AI 客户端连接到 spike.land 就能立即访问 80+ 生产就绪的工具。

无需定制集成。无需认证胶水代码。无需分页逻辑。工具就在那里，随时调用。

### 我们今天发布的 80+ 工具

我们的初始工具集涵盖五大类别：

**国际象棋竞技场** — 完整的竞技象棋，包含 ELO 评分、实时挑战和棋局状态管理。六个工具。`chess_new_game`、`chess_send_challenge`、`chess_make_move`、`chess_get_board`、`chess_get_elo`、`chess_resign`。

**图像工作室** — AI 驱动的图像分析、生成、增强、色彩提取、相册管理和流水线编排。42 个工具，基于可组合中间件架构构建，详见 [The Architecture of Scale](/blog/the-architecture-of-scale-building-tools)。

**代码空间** — 实时代码执行、转译和预览环境。五个工具，用于在隔离沙箱中运行 TypeScript 和 JavaScript。

**QA 工作室** — 基于 Playwright 的浏览器自动化。八个工具，用于脚本化端到端测试、截图捕获和交互录制。

**通用工具** — 网页获取、搜索、文件操作，以及一组不断增长的跨类别通用工具。

每个工具遵循相同的契约：Zod 类型化输入模式、叙事优先的响应、结构化 JSON 输出。[为什么要叙事式响应？](/blog/mcp-explained-universal-adapter-for-ai-tools#narrative-responses-tell-stories-not-dump-data) 因为 AI agent 不只需要数据——它需要能在上下文中推理的数据。

## API 网关：积分和 BYOK

我们的许多工具内部调用 AI 模型——图像分析、代码解释、内容生成。我们给你两种方式来支付这些模型调用，因为在你探索什么有效的时候，灵活性很重要。

**自带密钥 (BYOK)**

在仪表盘中连接你的 OpenAI、Anthropic 或 Replicate API 密钥。需要模型的工具调用通过你的密钥路由。你的成本。你的速率限制。完全控制。

**Spike.land 代理积分**

不想管理多个 API 账户？使用 spike.land 代理积分。一个积分池，统一计费，无需按服务开户。

- 1 积分 = 1 次标准模型 API 调用（近似值，因模型和 token 数量而异）
- Pro 和 Business 计划的积分按月滚动
- 额外积分 $0.10/积分

代理是透传的——我们在积分转换之外不加价。我们的利润来自平台和市场，不是模型推理。

## 技术栈

spike.land 完全运行在 Cloudflare Workers 上。无需配置服务器。无需选择区域。请求自动路由到最近的边缘节点。基础设施让开路，让工具来完成工作。

- **边缘 API**：Cloudflare Workers 上的 Hono 框架
- **MCP 注册中心**：Cloudflare Workers + D1（边缘 SQLite）
- **认证**：Better Auth + Drizzle ORM
- **前端**：Vite + React + TanStack Router
- **工具运行时**：按工具类别隔离的 Worker 环境

[完整架构](/docs/architecture)已文档化。我们公开构建，也将继续如此。

## 为什么选 MCP？

我们本可以构建 REST API。REST 大家都熟悉。人人都知道如何 curl 一个端点。

但 REST API 是为盯着文档的人类开发者设计的。它们优化的是紧凑 JSON、状态码、分页 token。AI agent 需要不同的契约——一种用自然语言告诉模型一个能力*做什么*、用类型化参数告诉它*怎么调用*、以及告诉它响应在上下文中*意味着什么*的契约。

MCP 就是这个契约。它把发现、类型化模式和叙事式响应定义为一等原语。当你的 AI agent 通过 MCP 连接到 spike.land 时，它不需要文档页面。它读取工具描述，推断正确的调用，然后执行。协议本身就是文档。

我们在 [MCP Explained](/blog/mcp-explained-universal-adapter-for-ai-tools) 中写了更多。简短版本：MCP 是一个表示层，不仅仅是传输层。它塑造的是模型如何*思考*你的能力，而不仅仅是如何*调用*它们。
